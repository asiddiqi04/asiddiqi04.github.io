Introduction

My goal for this website is to give a clear picture of what I worked on, what I learned, and how the term helped me grow as a software engineering student. I worked at TextNow during the Winter 2026 co-op term as a Full-Stack Software Developer Co-op, contributing to projects that supported internal teams and customer-focused product thinking.

About TextNow

TextNow is a communications company best known for helping people call and text through an app-based phone service. The company works in an area of computing that connects mobile apps, cloud services, data systems, internal platforms, and customer experience. During my term, I saw how software supports both the product users see and the internal systems employees rely on every day.

What stood out to me was the range of problems a software team can touch inside one company. Some work was close to the customer experience, while other work focused on making internal processes faster, clearer, and easier to manage.

Goals and Learning Outcomes

Build Useful Internal Tools

I wanted to improve at building software for real teams, not just for a class assignment. This goal was met through projects where I had to understand how employees worked, what information they needed, and how to make tools fit into existing processes.

Grow My Automation Skills

I wanted to learn how companies reduce repetitive work through automation. I made progress on this by working on feedback collection and onboarding workflows that moved information between systems with less manual effort.

Practice Clear Technical Communication

I wanted to get better at explaining technical work to people with different backgrounds. This goal became important when writing documentation, presenting product ideas, and turning project details into language stakeholders could understand.

Connect Software to Business Value

I wanted to understand how engineering decisions support business goals. The innovation challenge helped me see how customer feedback, product strategy, and implementation planning can come together in one proposal.

Looking back, I was most successful in improving my automation, communication, and product-thinking skills. I still want to keep growing in system design, especially around how larger services are maintained after the first version is built.

Job Description

As a Full-Stack Software Developer Co-op, my work involved building and improving internal software systems. Some tasks required technical implementation, while others required research, planning, documentation, and collaboration with people outside of engineering.

The most interesting part of the role was that the projects were practical and cross-functional. I needed programming and data skills, but I also had to ask good questions, understand user needs, and explain trade-offs. Some of these skills came from school, especially problem solving and software design, while many were strengthened on the job through real tools, real constraints, and real stakeholders.

Project 01

AI Tools Feedback Pipeline

Built a feedback system that helped teams see how internal AI tools were working for the people using them.

What I Built

  • Added simple feedback forms directly into internal tools so users could share their experience in the moment.
  • Created an automated flow that collected each response and stored it in one organized place.
  • Built a live dashboard so stakeholders could quickly review trends, issues, and improvement opportunities.
n8n Snowflake Streamlit Embedded HTML forms

Why it matters: The project gave the team one clear place to understand what was working, what needed attention, and how users actually felt about the tools.

Project 02

IT Automated Onboarding

Worked on a new employee onboarding system that reduces repetitive setup work and helps IT track each new hire from start to finish.

What I Built

  • Created the first phase of an automated process that checks for new hires and identifies who needs onboarding.
  • Set up automatic service desk tickets so IT has a clear record of each onboarding request.
  • Mapped employee information from the HR system to the fields needed for account creation.
n8n Dayforce API Jira Service Management API Active Directory / LDAP AWS Okta Microsoft Graph API

Why it matters: The project helps reduce manual handoffs, lowers the chance of missed setup steps, and gives IT a clearer audit trail for every new employee.

Project 03

Innovation Challenge - Smart Milestone Rewards

Contributed to a team pitch for a loyalty program that rewards long-term users and creates a more natural path toward paid features.

What I Built

  • Reviewed 22,993 customer feedback records to understand what users liked, disliked, and wanted from paid features.
  • Helped design a milestone reward path that gives users helpful in-app rewards as they continue using the product.
  • Outlined how the idea could be supported by a reliable software service and customer data.
  • Created a polished presentation deck using TextNow's brand style for the final pitch.
Enterpret Cypher Go gRPC Snowflake Figma Gamma

Why it matters: The pitch showed that many users already saw value in the product, and that a better reward experience could make paid options feel more timely and useful.

Conclusion

This work term showed me how software engineering creates value beyond writing code. The strongest projects started with a real need: a team needed clearer feedback, IT needed a more reliable onboarding process, and product stakeholders needed a better way to understand customer motivation. My role was to help turn those needs into systems, plans, and explanations that could be used by other people.

If someone were to summarize this report, I would want them to say that I learned how to connect technical work with human problems. I became more confident building practical tools, communicating with stakeholders, and thinking about software as part of a larger organization.
